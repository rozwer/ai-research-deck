Architecture · Summary 2026/SOLO/06

生成 · 検証 · 知識
の 3 層 + 安全層。

Claude がリーダーで段取りを握り、5 体のエージェント が分業。
為替で組んだループが、人流解析にもそのまま転用できている — 一度作れば再利用できる構造。

§ Generation Layer · 奔放に作る

候補を量産する側。
  • Claude (lead) — 段取り / 進捗管理 / 取りまとめ
  • Codex (impl) — コードを書く。同モデルで批判役兼任
  • Critic (red team) — わざと反対意見を出す
  • SubAgent ×N — 並列で先行研究調査・ルール遵守チェック

§ Verification Layer · 冷酷に叩く

数字だけで判定。AI を入れない。
  • 期間分け前進検証 — 学習 / 検証 を時系列で分離
  • 境界に空白 (embargo) — 情報漏洩防止
  • 多重試行ペナルティ — 偶然のいい数字を割り引く
  • 仮説事前登録 — データを見る前に確定

§ Knowledge Layer · 永続記憶

学んだことを次に渡す。
  • claude-mem — セッション越し状態 DB
  • Obsidian + 検索 Skill — 必要分だけ召喚
  • Skill 化 — 別プロジェクトに転用 (FX → 人流)
  • 朝の判断 で再利用 — ループが閉じる
Safety
3 layers
Layer I
物理ガード (Hooks)
削除はステージング行き。データ消失を物理的に止める。
Layer II
ネット制限 / 許可リスト
外部通信は許可リスト方式。許可外コマンドは即停止。
Layer III
機密の自動マスク
API キー / 個人情報を自動検出して伏字化。
⚠ これが無いと、
ある日 87 GB 全消去 される。

Case II · Stacked-slices · 2D → 3D 2026/NAGOYA/13
§ Theme 1 — 3D 建物復元 (deep dive)

地図も LiDAR も使わず、
GPS 軌跡だけから 建物の立体形状を起こす。

入力は 2D の点群と「ここに何かある」という事実だけ。出力は 3D 建物。
高度 (alt_hae) を縦方向に展開 → {4m / 10m / 15m / 25m} で水平にスライス → 各層の点密度から境界を抽出 → 連続層をポリゴンで包む — 5 段階で建物が立ち上がる。

▸ stacked-slices reconstruction stage 0 / 5
stage 0 · idle
地面だけがある状態。

入力は 553 万件の GPS 点と「ここに何かある」という事実だけ。地図も LiDAR も使えない。

stage 1 · GPS scatter
点群が降ってくる。

匿名 ID + 緯度経度 + 高度の点が地面に散る。これだけで街の形を当てる。

stage 2 · vertical poles
高度方向に伸ばす。

各点の高度 (alt_hae / alt_hat) を縦方向に展開。同じ場所でも上層・下層で違う動き — 建物の階層構造が垂直に現れる。

stage 3 · layer slicing
高度帯にスライス。

{ 4m, 10m, 15m, 25m } の帯で水平に切る。各層の点密度から建物の境界を抽出。

stage 4 · polygon walls
層をポリゴンで包む。

連続するスライス間を半透明の壁で繋ぐ。境界の不確実性は透明度で表現。

stage 5 · solid building
建物の形が立ち上がる。

壁を不透明にし、屋根を被せ、最終形状を確定。地図なし・LiDAR なしから街の 3D 構造が復元できる。

§ AI が手を動かしたところ — 553 万 GPS 点の前処理 (alt_hae の単位揃え) → 高度帯の自動決定 (KDE で峰検出) → 各層のα-shape ポリゴン化 → 不確実性付き輪郭描画。従来は LiDAR + 航空写真 + 3D 都市モデルで数千万円規模、博士課程テーマ級の難度。
Case II · City as a Circuit 2026/NAGOYA/14
§ Case II — 都市を回路として見る

住宅は電池。ビルはコンデンサ。
都市は 1 つの回路として、人流を予測式に落とす。

人 = 電流。住宅 = 夜間に充電し、昼間に放電する battery。オフィスビル = 一時的に滞留させる capacitor。道路・改札 = resistor。商業施設 = load (消費)。
この見立てで Kirchhoff の法則 (Σ I = 0) を立てれば、3 テーマがひとつの保存則に収まる。

§ Circuit · City of Nagoya as electrical loopI(t) = pedestrian flow
Predictive formΣ I_node = 0
I(t) = −dQ_residential/dt  +  C·dV_office/dt  +  ΔV / R_route  −  P_commercial / V
＝ 住宅の充放電 + オフィスの滞留 + 経路抵抗 − 商業の消費。3 テーマがひとつの保存則に統合される。
